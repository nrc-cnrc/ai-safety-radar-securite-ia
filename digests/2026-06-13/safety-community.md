## Notable GitHub Releases & Tools

**[InferenceWall v0.1.9](https://github.com/inferwall/inferwall/releases/tag/v0.1.9)** - This prompt injection detection and response system added an OpenAI-compatible scanning proxy that enables zero-code-change integration for existing applications. The release includes detection coverage improvements for vendor keys, data exfiltration attempts, and ML normalization techniques, plus constant-time API key comparison for security. This matters because it provides a practical deployment path for prompt injection defenses that can be adopted without modifying existing codebases.

**[Node9 Proxy v1.33.0](https://github.com/node9-ai/node9-proxy/releases/tag/v1.33.0)** - Added multi-agent cost tracking, drop-proof ingestion, egress control, and an outbox shipper, along with a security fix for destructive command de-obfuscation that closed a verified bypass in the command filtering system. The release focuses on operational reliability and security hardening for AI proxy deployments. This matters because it addresses critical infrastructure needs for secure AI deployment, particularly around cost control and command execution safety.

**[RobTM v0.1.0](https://github.com/AnwarDebes/RobTM/releases/tag/v0.1.0)** - A reference implementation of a robust Tsetlin Machine that provides per-sample certificates indicating the minimum number of feature flips needed to change a prediction. Unlike typical robustness work that only reports accuracy under attack, this provides formal guarantees for individual inputs. This matters because it represents a shift toward interpretable and certifiable robustness in machine learning, offering concrete bounds rather than statistical measures.

**[Glassbox v4.4.0](https://github.com/designer-coderajay/glassbox-mech/releases/tag/v4.4.0)** - Updated the interpretability framework with verbalizer sets, counterfactual verification capabilities, and evidence tiers for model explanation. The release focuses on making model decisions more transparent and verifiable through structured explanation mechanisms. This matters because it advances the state of explainable AI tooling, providing researchers and practitioners with better methods for understanding and validating model behavior.

**[ELUZA Safe AI Tool v0.1.0](https://github.com/DENCH49/eluza-safe-ai-tool/releases/tag/v0.1.0)** - A bilingual Thai/English offline safety classification tool that includes thought classification, safety boundaries, and access control for protected AI surfaces. The tool provides example safety scenarios and operates entirely offline for privacy. This matters because it demonstrates culturally-aware AI safety tooling and highlights the importance of developing safety systems that work across different languages and cultural contexts.
