## Notable GitHub Releases & Tools

**Humanbound v2.7.0** ([Release](https://github.com/humanbound/humanbound/releases/tag/v2.7.0)) - Added telemetry opt-out tracking and security improvements for handling redirects with authentication credentials. The release introduces a one-time `telemetry_disabled` event when users opt out via `hb telemetry disable` or environment variables, and prevents following HTTP redirects on requests carrying auth credentials to avoid credential leakage to untrusted hosts. This matters because it provides better visibility into user privacy preferences while strengthening security around credential handling in HTTP requests.

**Turnkey v0.1.0** ([Release](https://github.com/lijrjyan/turnkey/releases/tag/v0.1.0)) - A new "Bring Your Own Detector" framework for developing and debugging jailbreak detectors, guards, and safety policies for AI systems. It provides paired reference/intervention evaluation, typed model signals (logprobs, hidden states, gradients), inspectable event traces, and reproducible test harnesses. This matters because it addresses the critical need for robust tooling to develop and validate AI safety mechanisms as models become more capable.

**DeepSentry v2.0.2 Ultimate** ([Release](https://github.com/asaotomo/DeepSentry/releases/tag/DeepSentry_v2.0.2_Ultimate)) - A comprehensive security monitoring system with Agent runtime capabilities, network device compatibility, remote protocols, and improved Terminal User Interface (TUI) experience. The release includes Runtime v3 architecture, enhanced device type recognition, and security hardening for deployment workflows. This matters because it provides organizations with advanced capabilities for monitoring and securing AI agent deployments across diverse network infrastructures.

**Claude Code Privacy Guard v0.2.8** ([Release](https://github.com/datumbrain/claude-code-privacy-guard/releases/tag/0.2.8)) - Enhanced privacy protection tool for Claude Code with new allowlist management UI, redact/warn modes, and improved false positive handling. The release adds a web interface for managing allowlists, multiple prompt handling modes beyond blocking, and better support for legitimate use cases that might otherwise trigger false positives. This matters because it makes privacy protection more user-friendly and practical for developers while maintaining strong security boundaries.

**MLflow Model Metrics Sampling** - Implemented memory-bounded sampling for the `get-history-bulk-interval` API endpoint that powers Model metrics charts, preventing memory exhaustion on large datasets by intelligently sampling metric history rather than loading all points. This enables the UI to remain responsive when visualizing metrics from long-running experiments with extensive logging. This matters because it removes a significant scalability bottleneck that could make MLflow unusable for organizations with large-scale ML experiments.
