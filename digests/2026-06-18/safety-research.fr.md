# Articles de recherche (2026-06-18)

## Articles clés

**[TRAP: Benchmark for Task-completion and Resistance to Active Privacy-extraction](https://arxiv.org/abs/2606.18996v1)** évalue les agents opérant sur des documents sensibles, où ils doivent accomplir des tâches en utilisant des informations privées sans jamais les exposer dans leurs réponses. Le benchmark révèle des tensions fondamentales entre capacité et confidentialité, les modèles plus performants étant plus vulnérables aux attaques de confidentialité. Ceci souligne des défis de sécurité critiques pour le déploiement d'agents dans des flux de travail intensifs en documents où les violations de confidentialité pourraient avoir des conséquences graves.

**[PhantomSkill: Malicious Code Injection in Agent Skill Ecosystems](https://arxiv.org/abs/2606.19191v1)** démontre comment les attaquants peuvent dissimuler des comportements malveillants dans les compétences d'agents LLM en intégrant des vulnérabilités dans des ressources auxiliaires plutôt que dans des descriptions textuelles. La technique VulMask crée des implémentations en forme de vulnérabilité activées uniquement sous des conditions contrôlées par l'attaquant, contournant les méthodes de détection actuelles. Cela expose un vecteur d'attaque de chaîne d'approvisionnement significatif alors que les écosystèmes d'agents s'appuient de plus en plus sur des compétences tierces.

**[CodeSentinel: A Three-Layer Defense Against Indirect Prompt Injection in Code Contexts](https://arxiv.org/abs/2606.19235v1)** traite les attaques d'injection de prompt indirecte où des instructions malveillantes sont cachées dans les commentaires de code, les chaînes de caractères et les identifiants que les LLM de code récupèrent depuis des sources externes. La défense à trois couches combine un filtrage guidé par la syntaxe, un scoring dynamique et un élagage de nœuds pour assainir le contenu risqué tout en préservant la fonctionnalité. Ceci représente une infrastructure cruciale pour sécuriser les systèmes de génération de code alors qu'ils interagissent de plus en plus avec des dépôts de code non fiables.

**[Detecting Hidden ML Training With Zero-Overhead Telemetry](https://arxiv.org/abs/2606.19262v1)** évalue le monitoring basé sur le matériel des charges de travail GPU pour la gouvernance de l'IA, testant si les développeurs peuvent échapper aux mécanismes de détection en utilisant uniquement des signaux de télémétrie agnostiques au contenu. À travers 5 tours d'itération adversarielle, les systèmes de monitoring ont maintenu une haute précision malgré les tentatives d'évasion, suggérant que la gouvernance activée par le matériel pourrait être plus robuste que supposé précédemment.

**[OpenAnt: LLM-Powered Vulnerability Discovery Through Code Decomposition, Adversarial Verification, and Dynamic Testing](https://arxiv.org/abs/2606.19149v1)** présente un système automatisé de découverte de vulnérabilités qui décompose les grandes bases de code en unités analysables, utilise les LLM pour le raisonnement sémantique sur les problèmes de sécurité, et emploie une vérification adversarielle pour réduire les faux positifs. Le système adresse les limitations clés de l'analyse statique traditionnelle tout en gérant les défis de contexte et de vérification de l'application des LLM à l'analyse de sécurité à l'échelle du dépôt.

**[Beyond Safe Data: Pretraining-Stage Alignment with Regular Safety Reflection](https://arxiv.org/abs/2606.19168v1)** propose le Safety Reflection Pretraining pour traiter la façon dont les LLM peuvent composer des connaissances bénignes en comportements dangereux même lorsqu'ils sont entraînés sur des données sûres filtrées. La méthode injecte régulièrement des tokens de réflexion de sécurité pendant le pré-entraînement pour construire des capacités de raisonnement de sécurité directement dans les représentations fondamentales du modèle. Ceci représente un changement important des mesures de sécurité réactives vers l'intégration proactive de la sécurité pendant le développement du modèle.

**[Giskard: Byzantine Robust and Confidential Aggregation for Large-Scale Decentralized Learning](https://arxiv.org/abs/2606.19129v1)** s'attaque au défi de gérer simultanément la confidentialité et les comportements byzantins dans les systèmes d'apprentissage décentralisés. Le framework permet une agrégation robuste aux byzantins tout en préservant la confidentialité grâce à des techniques cryptographiques, comblant un écart critique dans l'apprentissage fédéré sécurisé où les participants ne peuvent pas être entièrement fiables.

**[PYPILINE: Malicious PyPI Package Detection via Suspicious API Knowledge and Agent Workflow](https://arxiv.org/abs/2606.19063v1)** introduit un système d'agent LLM pour détecter les paquets Python malveillants qui combine une base de connaissances d'API suspectes avec des workflows d'agent interprétables. Le système adresse les limitations des approches traditionnelles basées sur des règles et ML en fournissant des décisions de détection explicables et une meilleure adaptation aux nouveaux patterns d'attaque dans la chaîne d'approvisionnement logicielle.

**[Quantifying Compromise Risk in Exceptional Access Architectures Under Sparse and Indirect Evidence](https://arxiv.org/abs/2606.19106v1)** fournit la première analyse quantitative des risques de compromission dans les systèmes d'accès exceptionnel légal, distinguant entre les architectures de couche de transmission et de couche de plateforme. Utilisant des preuves éparses et indirectes en raison du manque de jeux de données de compromission publics, l'analyse révèle des profils de risque significativement différents entre les approches architecturales, informant les débats politiques critiques sur les portes dérobées de chiffrement.

**[AI Sandboxes: A Threat Model, Taxonomy, and Measurement Framework](https://arxiv.org/abs/2606.18532v1)** développe un framework complet pour évaluer les systèmes d'IA dans des environnements contrôlés, particulièrement pour l'IA physique et les systèmes cyber-physiques. Le framework adresse les défis d'assurance lorsque les systèmes d'IA interagissent avec des processus physiques, fournissant une infrastructure essentielle pour les tests et le déploiement sûrs de l'IA dans les applications critiques pour la sécurité.
