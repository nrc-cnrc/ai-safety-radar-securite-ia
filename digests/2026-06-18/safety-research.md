# Research Papers (2026-06-18)

## Key Papers

**[TRAP: Benchmark for Task-completion and Resistance to Active Privacy-extraction](https://arxiv.org/abs/2606.18996v1)** evaluates agents operating on sensitive documents, where they must complete tasks using private information while never exposing it in responses. The benchmark reveals fundamental tensions between capability and privacy, with more capable models being more vulnerable to privacy attacks. This highlights critical safety challenges for deploying agents in document-intensive workflows where privacy breaches could have severe consequences.

**[PhantomSkill: Malicious Code Injection in Agent Skill Ecosystems](https://arxiv.org/abs/2606.19191v1)** demonstrates how attackers can hide malicious behavior in LLM agent skills by embedding vulnerabilities in auxiliary resources rather than textual descriptions. The VulMask technique creates vulnerability-shaped implementations activated only under attacker-controlled conditions, bypassing current detection methods. This exposes a significant supply-chain attack vector as agent ecosystems increasingly rely on third-party skills.

**[CodeSentinel: A Three-Layer Defense Against Indirect Prompt Injection in Code Contexts](https://arxiv.org/abs/2606.19235v1)** addresses indirect prompt injection attacks where malicious instructions are hidden in code comments, strings, and identifiers that code LLMs retrieve from external sources. The three-layer defense combines syntax-guided filtering, dynamic scoring, and node pruning to sanitize risky content while preserving functionality. This represents crucial infrastructure for securing code generation systems as they increasingly interact with untrusted code repositories.

**[Detecting Hidden ML Training With Zero-Overhead Telemetry](https://arxiv.org/abs/2606.19262v1)** evaluates hardware-based monitoring of GPU workloads for AI governance, testing whether developers can evade detection mechanisms using only content-agnostic telemetry signals. Across 5 rounds of adversarial iteration, monitoring systems maintained high accuracy despite evasion attempts, suggesting that hardware-enabled governance may be more robust than previously assumed.

**[OpenAnt: LLM-Powered Vulnerability Discovery Through Code Decomposition, Adversarial Verification, and Dynamic Testing](https://arxiv.org/abs/2606.19149v1)** presents an automated vulnerability discovery system that decomposes large codebases into analyzable units, uses LLMs for semantic reasoning about security issues, and employs adversarial verification to reduce false positives. The system addresses key limitations of traditional static analysis while managing the context and verification challenges of applying LLMs to repository-scale security analysis.

**[Beyond Safe Data: Pretraining-Stage Alignment with Regular Safety Reflection](https://arxiv.org/abs/2606.19168v1)** proposes Safety Reflection Pretraining to address how LLMs may compose benign knowledge into unsafe behaviors even when trained on filtered safe data. The method regularly injects safety reflection tokens during pretraining to build safety reasoning capabilities directly into the model's foundational representations. This represents an important shift from reactive safety measures to proactive safety integration during model development.

**[Giskard: Byzantine Robust and Confidential Aggregation for Large-Scale Decentralized Learning](https://arxiv.org/abs/2606.19129v1)** tackles the challenge of simultaneously handling confidentiality and Byzantine behaviors in decentralized learning systems. The framework enables Byzantine-robust aggregation while preserving privacy through cryptographic techniques, addressing a critical gap in secure federated learning where participants cannot be fully trusted.

**[PYPILINE: Malicious PyPI Package Detection via Suspicious API Knowledge and Agent Workflow](https://arxiv.org/abs/2606.19063v1)** introduces an LLM-agent system for detecting malicious Python packages that combines a suspicious API knowledge base with interpretable agent workflows. The system addresses limitations of traditional rule-based and ML approaches by providing explainable detection decisions and better adaptation to novel attack patterns in the software supply chain.

**[Quantifying Compromise Risk in Exceptional Access Architectures Under Sparse and Indirect Evidence](https://arxiv.org/abs/2606.19106v1)** provides the first quantitative analysis of compromise risks in lawful exceptional access systems, distinguishing between transmission-layer and platform-layer architectures. Using sparse, indirect evidence due to the lack of public compromise datasets, the analysis reveals significantly different risk profiles between architectural approaches, informing critical policy debates about encryption backdoors.

**[AI Sandboxes: A Threat Model, Taxonomy, and Measurement Framework](https://arxiv.org/abs/2606.18532v1)** develops a comprehensive framework for evaluating AI systems in controlled environments, particularly for physical AI and cyber-physical systems. The framework addresses assurance challenges when AI systems interact with physical processes, providing essential infrastructure for safe AI testing and deployment in safety-critical applications.
