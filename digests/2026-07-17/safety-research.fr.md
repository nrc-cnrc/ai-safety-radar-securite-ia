# Articles de recherche (2026-07-17)

## Articles clés

**[Pretraining Data Can Be Poisoned through Computational Propaganda](https://arxiv.org/abs/2607.15267v1)** examine comment les adversaires peuvent introduire des comportements nuisibles dans les grands modèles de langage en empoisonnant les données de pré-entraînement à grande échelle par le biais de comportements coordonnés inauthentiques, exploitant l'interaction entre le contenu empoisonné et les pipelines de curation des données. Ceci représente une avancée critique dans la compréhension des menaces réelles d'empoisonnement de données au-delà des environnements de laboratoire.

**[Transcoders for Investigating Deception in Language Models](https://arxiv.org/abs/2607.14791v1)** démontre comment les transcodeurs (transcodeurs par couche) peuvent être utilisés pour l'interprétabilité mécanistique des comportements trompeurs dans les modèles de langage, construisant des graphiques d'attribution qui capturent les activations de caractéristiques et les dépendances inter-caractéristiques pour analyser les comportements critiques pour la sécurité. Ceci fournit de nouveaux outils pour comprendre et potentiellement détecter la tromperie au niveau des circuits.

**[MemPoison: Uncovering Persistent Memory Threats and Structural Blind Spots in LLM Agents](https://arxiv.org/abs/2607.14651v1)** introduit un benchmark complet révélant comment la mémoire persistante dans les agents LLM crée de nouvelles surfaces d'attaque où du contenu adversarial peut être injecté par des interactions standard et persister à travers les sessions pour déformer les comportements futurs. Ceci met en évidence un défi de sécurité fondamental alors que les agents deviennent plus persistants en mémoire.

**[BadWAM: When World-Action Models Dream Right but Act Wrong](https://arxiv.org/abs/2607.15207v1)** expose un mode de défaillance critique dans les modèles monde-action où le couplage entre la génération d'actions et la prédiction du monde futur—souvent considéré comme une source de sécurité—peut en fait être fragile, avec des modèles potentiellement maintenant des prédictions correctes du monde tout en générant des actions dangereuses. Ceci remet en question les hypothèses sur les bénéfices de sécurité inhérents des approches basées sur les modèles du monde.

**[Context Contamination in LLM Analysis of Network Security Logs: Poison with Passive Prompt Injection and Mitigation Evaluation](https://arxiv.org/abs/2607.14493v1)** démontre comment les LLM déployés dans les centres d'opérations de sécurité font face à une nouvelle vulnérabilité où les adversaires peuvent intégrer des charges utiles d'injection de prompt dans les champs générateurs de logs qui persistent dans le stockage et s'exécutent lorsque les logs sont traités comme contextes de langage naturel. Ceci représente une nouvelle classe d'attaques de chaîne d'approvisionnement ciblant les outils de sécurité basés sur les LLM.

**[DataShield: Uncovering Risky Fine-Tuning Data Across LLMs Through Consensus Subspace Alignment](https://arxiv.org/abs/2607.15081v1)** développe des méthodes pour identifier les données de fine-tuning qui affaiblissent substantiellement les capacités de sécurité des LLM en utilisant l'alignement de sous-espaces de consensus à travers plusieurs modèles, abordant le problème critique que même le fine-tuning bénin spécifique aux tâches peut compromettre les garde-fous de sécurité. Ceci fournit des outils pratiques pour la curation de données dans les flux de travail de fine-tuning soucieux de la sécurité.

**[When Words Are Safe But Actions Kill: Probing Physical Danger Beyond Text Safety in Hidden-State Risk Space](https://arxiv.org/abs/2607.15218v1)** révèle par l'analyse des états cachés que le danger du contenu et le danger physique forment des signaux séparables dans les représentations des LLM, montrant que des instructions linguistiquement bénignes peuvent devenir dangereuses lorsqu'elles sont ancrées dans des contextes physiques incarnés. Cette découverte est cruciale pour développer des mesures de sécurité pour les systèmes d'IA incarnée.

**[Setup Complete, Now You Are Compromised: Weaponizing Setup Instructions Against AI Coding Agents](https://arxiv.org/abs/2607.15143v1)** présente la première évaluation systématique d'attaques de chaîne d'approvisionnement où les agents de codage IA peuvent être compromis simplement en lisant de la documentation malveillante qui les redirige vers l'installation de dépendances non fiables ou de packages vulnérables. Ceci démontre comment la documentation devient un vecteur d'exécution de code dans les flux de travail de développement automatisés.

**[Bad Memory: Evaluating Prompt Injection Risks from Memory in Agentic Systems](https://arxiv.org/abs/2607.14611v1)** étudie comment la mémoire persistante dans les systèmes agentiques crée de nouvelles surfaces d'attaque où des instructions malveillantes intégrées dans les fichiers de mémoire peuvent influencer le comportement futur des agents, montrant que les agents basés sur la mémoire sont vulnérables aux attaques d'injection de prompt indirectes qui persistent à travers les sessions. Cette recherche est essentielle alors que les agents s'appuient de plus en plus sur un état persistant pour améliorer leur fonctionnalité.

**[Evaluating Epistemic Uncertainty: Beyond OOD Detection and Active Learning](https://arxiv.org/abs/2607.14817v1)** remet en question les méthodes d'évaluation actuelles pour l'incertitude épistémique en montrant que les stratégies Bayes-optimales pour les tâches communes ne s'alignent pas avec les scores de quantification d'incertitude standard, proposant une évaluation basée sur la capacité à identifier le regret (erreur réductible) à la place. Ce travail a des implications importantes pour développer une estimation d'incertitude fiable dans les applications critiques pour la sécurité.
