# Research Papers (2026-07-17)

## Key Papers

**[Pretraining Data Can Be Poisoned through Computational Propaganda](https://arxiv.org/abs/2607.15267v1)** investigates how adversaries can introduce harmful behaviors into large language models by poisoning pretraining data at scale through coordinated inauthentic behavior, exploiting the interaction between poisoned content and data curation pipelines. This represents a critical advancement in understanding real-world data poisoning threats beyond laboratory settings.

**[Transcoders for Investigating Deception in Language Models](https://arxiv.org/abs/2607.14791v1)** demonstrates how transcoders (per-layer transcoders) can be used for mechanistic interpretability of deceptive behaviors in language models, constructing attribution graphs that capture feature activations and inter-feature dependencies to analyze safety-critical behaviors. This provides new tools for understanding and potentially detecting deception at the circuit level.

**[MemPoison: Uncovering Persistent Memory Threats and Structural Blind Spots in LLM Agents](https://arxiv.org/abs/2607.14651v1)** introduces a comprehensive benchmark revealing how persistent memory in LLM agents creates new attack surfaces where adversarial content can be injected through standard interactions and persist across sessions to distort future behavior. This highlights a fundamental security challenge as agents become more memory-persistent.

**[BadWAM: When World-Action Models Dream Right but Act Wrong](https://arxiv.org/abs/2607.15207v1)** exposes a critical failure mode in world-action models where the coupling between action generation and future world prediction—often viewed as a source of safety—can actually be fragile, with models potentially maintaining correct world predictions while generating unsafe actions. This challenges assumptions about the inherent safety benefits of world-model-based approaches.

**[Context Contamination in LLM Analysis of Network Security Logs: Poison with Passive Prompt Injection and Mitigation Evaluation](https://arxiv.org/abs/2607.14493v1)** demonstrates how LLMs deployed in Security Operations Centers face a novel vulnerability where adversaries can embed prompt injection payloads in log-generating fields that persist in storage and execute when logs are processed as natural language contexts. This represents a new class of supply-chain attacks targeting LLM-based security tools.

**[DataShield: Uncovering Risky Fine-Tuning Data Across LLMs Through Consensus Subspace Alignment](https://arxiv.org/abs/2607.15081v1)** develops methods for identifying fine-tuning data that substantially weakens LLM safety capabilities by using consensus subspace alignment across multiple models, addressing the critical problem that even benign task-specific fine-tuning can compromise safety guardrails. This provides practical tools for data curation in safety-conscious fine-tuning workflows.

**[When Words Are Safe But Actions Kill: Probing Physical Danger Beyond Text Safety in Hidden-State Risk Space](https://arxiv.org/abs/2607.15218v1)** reveals through hidden-state analysis that content danger and physical danger form separable signals in LLM representations, showing that linguistically benign instructions can become unsafe when grounded in physical embodied contexts. This finding is crucial for developing safety measures for embodied AI systems.

**[Setup Complete, Now You Are Compromised: Weaponizing Setup Instructions Against AI Coding Agents](https://arxiv.org/abs/2607.15143v1)** presents the first systematic evaluation of supply-chain attacks where AI coding agents can be compromised simply by reading malicious documentation that redirects them to install untrusted dependencies or vulnerable packages. This demonstrates how documentation becomes a vector for code execution in automated development workflows.

**[Bad Memory: Evaluating Prompt Injection Risks from Memory in Agentic Systems](https://arxiv.org/abs/2607.14611v1)** studies how persistent memory in agentic systems creates new attack surfaces where malicious instructions embedded in memory files can influence future agent behavior, showing that memory-based agents are vulnerable to indirect prompt injection attacks that persist across sessions. This research is essential as agents increasingly rely on persistent state for improved functionality.

**[Evaluating Epistemic Uncertainty: Beyond OOD Detection and Active Learning](https://arxiv.org/abs/2607.14817v1)** challenges current evaluation methods for epistemic uncertainty by showing that Bayes-optimal strategies for common tasks don't align with standard uncertainty quantification scores, proposing evaluation based on the ability to identify regret (reducible error) instead. This work has important implications for developing reliable uncertainty estimation in safety-critical applications.
